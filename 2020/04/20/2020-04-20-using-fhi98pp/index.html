    Using FHI98PP program to generate pseudopotentials
    2020/04/20 Research PseudoPotential fhi98pp

    Here I write the workflow to generate pseudopotentials with FHI98PP program for personal reference.

    Installation

    • Download the code and decompress

    • Change to the root directory of fhi98pp

    • In lib/make. and src/Makefile change the variable FC to the fortran compiler that you have, for me, the intel fortran compiler ifort is used

    • Then compile by typing

    • In fhi98pp/work, we can see the executable psgen that will be used to generate pseudopotentials.
      Before running the code, don’t forget to set up environmental variable,

    • Unfortunately, I see wiered runtime error on midway even though I can successfully compile the code. So, I can only run this code on my Mac.

    Generate psedopotentials

    FHI98PP has its own defined input format and will be explained later.
    The good news is ABINIT website provides the input files for most elemets and you can find them here: LDA functional and
    GGA functional.
    Notes:

    1. In addition to fhi98pp input files, the two websites also provide generated psuedopotentials in fhi/pspnc formats. These fhi/pspnc files are NOT necessarily generated by the fhi98pp inputs.
    2. When you generate pseudopotentials from the fhi98pp inputs, probably you want to modify the inputs to meet your requirement.

    Format of fhi98pp inputs

    This is a sample input (06-C.GGA.ini) for fhi98pp to generate Troullier-Martins type pseudopotential for carbon.

    Here are the line-by-line explanation.

    • The first line contains 5 variables.

      variable description
      z nuclear charge
      nc number of core orbitals
      nv number of valence orbitals
      iexc index of exchange-correlation functional
      rnlc using linearized/nonlinearized exchange-correlation

      Note: rnlc = 0 means linearized core-valence exchange-correlation is used.
      Note: iexc must be chosen from the following,

      ixc Functional
      1 LDA Wigner
      2 LDA Hedin/Lundquist
      3 LDA Perdew/Zunger
      4 GGA Perdew/Wang
      5 GGA Becke, Perdew
      6 GGA PBE
      7
      8 LDA Perdew/Wang
      9 GGA BLYP
      10 GGA Perdew/Wang, Lee/Yang/Paar

      Note: the index of xc in fhi98pp is different from that in ABINIT.
      When we try to use the pseudopotentials generated by fhi98pp in ABINIT,
      we have to look up the index of xc in the manual of ABINIT.

    • Line 2 to 4 in this sample input represent the electronic structure of diamond.

      variabl description
      n principle quantum number
      l angular momentum quantum number
      f occupation number

      So, in this example the electronic structure of diamond is $1s^22s^22p^2$.

    • Next line has two parts, the max angular quantum number to be considered (lmax) and the type of pseudopotential to generate ( s_pp_def ).
      In this case, the largest quantum number to be considered is 2 ( d-type orbital ) and the type is “t” ( Troullier-Martins type ).

    • Next several lines are optional.

      variable description
      lt For lt-th pseudopotential compotent: use alternate cutoff radius and/or reference energy and/or scheme from present line
      rct cutoff radius, = 0 ( using default ) or > 0 ( user-defined cutoff radius )
      et reference energy, = 0 ( using default ) or $\neq 0$ ( user-defines reference energy )
      s_pp_type type of the orbitals of this angular momentum

    Run the program and generate pseudopotential

    To generate the pseduopotential, run the executable psgen under fhi98pp_root/work

    The executable produces a pseudopotential named C.pbe-mt.cpi.

    Converting pseudopotentials

    After previous steps, we have a pseudopotential in cpi format. However, this file cannot be
    directly used in ABINIT or Quantum Espresso. So, we have to convert the format to ABINIT’s fhi format and then Quantum Espresso’s upf format.

    ABINIT format (fhi)

    This step is trivial, we just need to add 7 lines to the beginning of the cpi file.

    • Line 1, not important, sumply notes about the pseudopotential. e.g.
      Diamond, fhi98pp: Troullie-Martins type, GGA PBE

    • Line 2 contains three varaibles

      variable description
      zatom the atomic number, i.e. the index of the element in periodic table
      zion the ionic charge, i.e. the number of valence electrons
      pspdat the date of pseudopotential generation

    • Line 3 contains

      variable description
      pspcod format of the pseudopotential, must be 6, because the format will be fhi
      pspxc index of exchange-correlation functional. Note: the index in abinit is different from fhi98pp. Checkout ixc
      lmax the maximal angular momentum described in the pseudopotential, the same as in fhi98pp input
      lloc the angular momentum of the potential to be used as local pseudopotential
      mmax the number of radial mesh points, first number on the 12th line in original cpi file
      r2well always set to zero

    • Line 4 contains

      variable description
      rchrg radius at which the core charge vanish, 0.0 if not specified otherwise in fhi98pp input
      fchrg amplitude of core charge, 0.0 if not core-corretion performed
      qchrg integrated model core charge, 0.0, will not be used by ABINIT

    • Line 5-7 are meaningless, simply copy the following

    • Copy everything from the cpi file and append to the end of fhi file. NO EMPTY LINE!!!

    Quantum Espresso format (upf)

    After converting the pseudopotential to fhi format, we can convert the fhi file to Quantum Espresso’s upf format by using the executable under QE_ROOT/upftools/fhi2upf.x.
    Simply run it in command line and it will ask for the information it needs.
